direction_id

In the trips.txt file there is a direction_id field. This document addresses MARTA's implementation of that field.

Google's specifications only allow the use of 0 or 1 in this field, so it would have been impossible for MARTA to establish 4 different values for the eastbound/​westbound/​northbound/​southbound directional indicators that are commonly used with MARTA routes. One might have wished for MARTA to assign 0 consistently to one non-intersecting pair (for example, northbound and eastbound) and 1 to the opposites of that pair (southbound and westbound) -- but MARTA's GTFS feed does not apply such regularity. I have not been able to identify any kind of regular pattern, so I put together this page to decrypt the usage in the MARTA GTFS feed.

The following table associates the direction_id field with the compass directions used by MARTA. I kept the headsign field as an aide to others in identifying mistakes I may have made. Please bring any such mistakes to my attention at joe@joederose.us, and I'll update this file promptly.

Asssuming I have it correct, a text file suitable for import into your database can be downloaded at direction.txt. Note that this file omits the route_id for the following reasons:

  • The data can just as easily be linked by the route_short_name field.
  • The route_id changes when MARTA updates its transit feed, so this table would require a full update for each successive update from MARTA if that field were included; as it is now, this table will require only minor updates, and will generally work even if this table and the MARTA feed are not of the same version.
