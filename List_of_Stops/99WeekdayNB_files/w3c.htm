The server returned the following status:

404 : C:\Trapeze\Trapeze6\INFOWEB\Hw\apps\system\css\w3c.css was not found.

Make sure you specified the File service in the URI and the correct path of the file.

Correct format for the URI path sent to the Trapeze Server Shell when requesting files is:

/ProfileName/File/Path/filename.ext

The /ProfileName part is optional. If not specified, the default profile is assumed. If specified, the ProfileName must match the profile name the the server is configured to use. /File is the name of the service that serves files. /Path is the path of the file (which may consist of more than one part.)
